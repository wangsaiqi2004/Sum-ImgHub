> 日期：2026-05-14  
> 状态：Draft

## 1. 产品概述

### 1.1 背景与动机

当前 SumAPI 生图工作台的 Studio 模式以**列表流**形式展示生成结果——用户提交提示词后，图片按时间线性排列在记录流中。这种模式适合批量生产，但在以下场景中存在局限：

- **迭代式创作**：用户想从一张图出发，反复优化、分支探索不同方向，列表流无法体现图片之间的「演化关系」。
- **多图比较**：用户想把多张候选图并排放在一起对比，列表流只能上下滚动。
- **空间化思维**：设计师习惯在画板上自由摆放素材、建立视觉关联，线性流程无法承载这种工作方式。

无限画布模式（Canvas Mode）将生成结果放入一个可自由平移、缩放的二维空间，用户可以在画布上**生成、摆放、选中、优化**图片，形成空间化的创作工作流。

### 1.2 产品目标

1. 提供一个无限画布工作区，用户可在画布上自由生成和排列 AI 图片。
2. 支持选中画布上的图片作为参考图，补充提示词后生成优化版本，形成「迭代树」。
3. 复用现有的 API 配置（API URL、API Key、模型选择），不新增后端接口。
4. 交互范式参考 Lovart 的 ChatCanvas：左侧无限画布 + 右侧生成面板。

### 1.3 核心设计原则

| 原则 | 说明 |
|------|------|
| **空间即记忆** | 生成的图片停留在画布上的用户摆放位置，空间关系本身就是创作记录 |
| **选中即上下文** | 选中一张图片，它自动成为下一次生成的参考图，无需额外操作 |
| **本地优先** | 画布状态（节点位置、连接关系）存储在 IndexedDB，不上传服务端 |
| **渐进复杂** | 初始界面只有画布 + 提示词输入框，高级功能通过选中图片后渐进展开 |

## 2. 信息架构

### 2.1 入口

在现有顶部导航栏中，新增「画布」入口，与「首页」「工作台」「广场」并列。

```
首页 | 工作台 | 画布 | 广场
```

URL hash: `#canvas`

`AppPage` 类型扩展为 `"home" | "studio" | "square" | "admin" | "canvas"`。

### 2.2 页面结构

```
┌──────────────────────────────────────────────────────────┐
│  顶栏：Logo · 导航（首页/工作台/画布/广场）· 连接状态    │
├────────────────────────────────────┬─────────────────────┤
│                                    │                     │
│                                    │   右侧面板          │
│         无限画布区域                │   (可折叠)          │
│         (占页面主体)                │                     │
│                                    │  - 提示词输入        │
│                                    │  - 参数配置          │
│                                    │  - 生成按钮          │
│                                    │  - 参考图预览        │
│                                    │                     │
├────────────────────────────────────┴─────────────────────┤
│  底部工具栏（浮动）：缩放控制 · 适应画布 · 小地图切换     │
└──────────────────────────────────────────────────────────┘
```

### 2.3 核心数据结构

```typescript
type CanvasNode = {
  id: string;
  type: "image";
  x: number;                    // 画布坐标
  y: number;
  width: number;                // 显示尺寸（画布逻辑像素）
  height: number;
  imageDataUrl?: string;        // 生成结果（存 IndexedDB，运行时 objectUrl）
  objectUrl?: string;           // 运行时 blob URL
  prompt: string;               // 生成时使用的提示词
  model: string;                // 使用的模型
  params: ImageParams;          // 生成参数快照
  status: "generating" | "success" | "error";
  error?: string;
  parentId?: string;            // 如果是从另一张图优化而来，记录父节点 ID
  referenceNodeId?: string;     // 参考图来源节点 ID
  createdAt: number;
  duration?: number;            // 生成耗时 ms
  imageWidth?: number;          // 原始图片像素宽
  imageHeight?: number;         // 原始图片像素高
};

type CanvasEdge = {
  id: string;
  fromNodeId: string;           // 父图节点
  toNodeId: string;             // 优化后的子图节点
};

type CanvasState = {
  nodes: CanvasNode[];
  edges: CanvasEdge[];
  viewport: {
    x: number;                  // 视口偏移
    y: number;
    zoom: number;               // 缩放倍率 0.1 ~ 3.0
  };
  selectedNodeId: string | null;
  lastSavedAt: number;
};
```

## 3. 画布交互设计

### 3.1 画布导航

#### 3.1.1 平移（Pan）

| 操作方式 | 触发条件 | 行为 |
|----------|----------|------|
| 鼠标中键拖拽 | 按住中键 + 移动 | 平移画布 |
| 空格 + 左键拖拽 | 按住空格 + 左键按下并移动 | 平移画布（Photoshop 习惯） |
| 双指滑动 | 触摸板两指移动 | 平移画布 |
| 右键拖拽 | 按住右键 + 移动 | 平移画布 |

**视觉反馈**：
- 平移时鼠标光标变为 `grab`（空格按下时）或 `grabbing`（拖拽中）。
- 画布背景显示点阵网格（`#e5e5e5` 圆点，间距 20px），随平移和缩放联动，提供空间感知。

#### 3.1.2 缩放（Zoom）

| 操作方式 | 触发条件 | 行为 |
|----------|----------|------|
| 滚轮 | Ctrl/Cmd + 滚轮 | 以鼠标位置为中心缩放 |
| 触摸板捏合 | 双指捏合/展开 | 以手势中心缩放 |
| 快捷键 Cmd/Ctrl + `=` | 键盘 | 放大一级（×1.2） |
| 快捷键 Cmd/Ctrl + `-` | 键盘 | 缩小一级（÷1.2） |
| 快捷键 Cmd/Ctrl + `0` | 键盘 | 重置为 100% |
| 底部缩放滑块 | 拖动 | 连续缩放 |
| 适应画布按钮 | 点击 | 自动缩放并平移，使所有节点适应当前视口 |

**缩放范围**：10% ~ 300%，以 5% 为步进。

**缩放行为**：
- 缩放始终以鼠标/手势位置为锚点（非视口中心），确保用户关注区域保持稳定。
- 缩放过程中节点内的图片使用合适的缩放渲染：zoom < 30% 时显示缩略图，zoom > 100% 时显示原图。
- 缩放到 zoom < 20% 时，节点仅显示色块占位 + 状态图标，不渲染图片细节。

#### 3.1.3 小地图（Minimap）

- 位于画布左下角，默认折叠为一个图标按钮。
- 点击展开后显示一个 180×120px 的半透明面板，其中：
  - 所有节点以小矩形表示。
  - 当前视口范围用高亮边框标注。
  - 可直接在小地图上点击/拖拽来快速跳转视口。
- 快捷键 `M` 切换小地图显示/隐藏。

### 3.2 节点操作

#### 3.2.1 选中（Select）

| 操作 | 行为 |
|------|------|
| 左键单击节点 | 选中该节点，取消其他选中。显示选中高亮边框（主题色 `#10B981` 2px 实线）+ 操作浮层 |
| 左键单击画布空白处 | 取消所有选中 |
| Escape 键 | 取消所有选中 |

**选中状态视觉**：
- 节点显示 2px 主题色（`#10B981`）边框。
- 节点上方出现**浮动操作栏**（Floating Toolbar），包含常用操作按钮。
- 右侧面板切换到「优化模式」，自动加载选中图片作为参考图预览。

#### 3.2.2 移动（Move）

| 操作 | 行为 |
|------|------|
| 左键拖拽已选中节点 | 移动节点到新位置 |
| 拖拽中按住 Shift | 锁定水平/垂直方向移动 |

**移动视觉反馈**：
- 拖拽时节点提升到最顶层（z-index 置顶）。
- 节点显示轻微阴影提升（`box-shadow: 0 8px 32px rgba(0,0,0,0.15)`）。
- 释放后节点落到目标位置，阴影恢复正常。
- 如果该节点有连接线（edge），连接线实时跟随。

#### 3.2.3 删除（Delete）

| 操作 | 行为 |
|------|------|
| 选中节点后按 Delete / Backspace | 弹出确认对话框："确认删除此图片？此操作不可撤销。" |
| 浮动操作栏中的删除按钮 | 同上 |

**删除行为**：
- 删除节点的同时删除该节点相关的所有 edge。
- 不级联删除子节点（子节点的 `parentId` 保留，但 edge 被移除）。
- 删除后画布自动保存状态到 IndexedDB。

#### 3.2.4 缩放节点尺寸（Resize）

- 选中节点后，四个角出现圆形拖拽手柄（8px 白色圆点 + 1px 边框）。
- 拖拽手柄可按比例缩放节点的**显示尺寸**（不影响图片实际像素）。
- 按住 Shift + 拖拽手柄可自由缩放（不锁定比例）。
- 最小显示尺寸：80×80px（画布逻辑像素）。

### 3.3 浮动操作栏（Floating Toolbar）

选中节点后，在节点上方居中显示一个浮动操作栏：

```
┌──────────────────────────────────────────────────┐
│  ✨优化  │  📥下载  │  📋复制提示词  │  🗑删除   │
└──────────────────────────────────────────────────┘
```

**定位规则**：
- 默认在节点上方 12px 处居中对齐。
- 如果节点顶部超出视口，操作栏改为节点下方 12px 处显示。
- 操作栏始终保持在画布视口内可见。

**各按钮行为**：

| 按钮 | 图标 | 行为 |
|------|------|------|
| 优化 | WandSparkles | 进入优化模式：右侧面板切换到优化表单，当前图片自动作为参考图 |
| 下载 | Download | 将当前节点的图片以 `{model}-{size}-{timestamp}.png` 格式下载 |
| 复制提示词 | Copy | 将节点的 prompt 复制到剪贴板，显示 toast 提示「已复制」 |
| 删除 | Trash2 | 弹出删除确认对话框 |

## 4. 生成流程

### 4.1 自由生成（无参考图）

**用户路径**：

```
用户在画布空白处 → 右侧面板处于「生成模式」 → 输入提示词 → 选择模型/参数 → 点击生成
```

**详细步骤**：

1. **取消选中**：用户点击画布空白处，或按 Escape，确保没有选中任何节点。此时右侧面板显示为**「生成模式」**。

2. **输入提示词**：在右侧面板的提示词输入框中输入描述。输入框为多行 textarea，支持 Shift+Enter 换行，Enter 提交（与 Studio 模式一致）。

3. **选择模型**：面板中显示当前选中的模型（继承自全局 API 配置）。用户可点击模型选择器切换模型。**支持的模型列表来自已验证的 API 连接**，与 Studio 模式共享同一份模型列表和 API Key。

4. **设置参数**：
   - 宽高比选择（下拉菜单，选项根据模型动态变化）
   - 分辨率选择（1K / 2K / 4K，根据模型是否为 pro 动态启用）
   - 质量选择（auto / low / medium / high）
   - 生成数量：画布模式下**固定为 1 张**（每次生成一个节点），不支持批量

5. **点击生成**：
   - 在画布上创建一个新节点，位置规则：
     - 如果画布为空：放置在视口中央。
     - 如果画布有节点：放置在当前视口中央（用户可视区域的中心）。如果该位置与已有节点重叠，向右偏移 `节点宽度 + 40px` 间距。
   - 节点立即显示为「生成中」状态。
   - 视口自动平移，确保新节点在可视区域内。

6. **生成中状态**：
   - 节点显示加载骨架屏（灰色渐变动画背景）。
   - 节点中央显示旋转加载图标 + "生成中..." 文字。
   - 节点下方显示计时器（已用时间）。
   - 右侧面板提示词输入框可以继续编辑（不阻塞用户操作）。
   - 用户可以在等待期间平移画布、查看其他节点。

7. **生成成功**：
   - 骨架屏被生成的图片替换（带 fade-in 动画，300ms）。
   - 节点尺寸根据图片宽高比自动调整（长边 = 280px 画布逻辑像素）。
   - 节点左下角显示模型名称 badge。
   - 节点右下角显示生成耗时。

8. **生成失败**：
   - 节点显示错误状态：红色边框 + 错误图标 + 错误消息摘要。
   - 浮动操作栏变为：「重试」「复制错误」「删除」。
   - 点击「重试」使用相同参数重新生成（替换当前节点内容）。

### 4.2 优化生成（以选中图为参考图）

**用户路径**：

```
用户选中一张成功的图 → 浮动操作栏点击「优化」→ 右侧面板切换到优化模式 → 输入补充提示词 → 点击生成优化
```

**详细步骤**：

1. **选中图片**：用户在画布上左键单击一张已成功生成的图片节点。

2. **点击优化按钮**：
   - 途径一：点击浮动操作栏的「优化」按钮。
   - 途径二：双击节点直接进入优化模式。
   - 途径三：右侧面板在选中状态下也会显示「基于此图优化」引导入口。

3. **右侧面板切换到「优化模式」**：

   面板内容变化：

   ```
   ┌─────────────────────────────┐
   │  ← 返回自由生成              │
   │                             │
   │  📷 参考图预览               │
   │  ┌─────────────────────┐    │
   │  │  [选中图的缩略图]     │    │
   │  │  1024x1024 · 126KB   │    │
   │  └─────────────────────┘    │
   │  已自动压缩为参考图           │
   │                             │
   │  原始提示词（只读展示）：     │
   │  "a cute orange cat..."     │
   │                             │
   │  补充优化提示词：             │
   │  ┌─────────────────────┐    │
   │  │                     │    │
   │  │  (textarea)         │    │
   │  │                     │    │
   │  └─────────────────────┘    │
   │  placeholder:               │
   │  "描述你想要的修改，如：      │
   │   改为夜景、增加雪花效果..."  │
   │                             │
   │  模型：[gpt-image-2-pro ▼]  │
   │  宽高比：[继承原图 ▼]       │
   │  分辨率：[继承原图 ▼]       │
   │                             │
   │  [════ 生成优化 ════]        │
   │                             │
   └─────────────────────────────┘
   ```

4. **参考图自动压缩**：
   - 选中图片自动压缩为参考图格式：
     - 最长边压缩到 1024px。
     - 格式转为 WebP。
     - 质量 0.85。
     - 压缩后大小限制 ≤ 1MB，超过则继续降低质量直到满足。
   - 压缩结果缓存在内存中（避免重复压缩）。
   - 面板中显示压缩后的尺寸和大小。

5. **参数继承与覆盖**：
   - **模型**：默认继承原图使用的模型，用户可手动切换。
   - **宽高比**：默认继承原图的宽高比。下拉菜单新增「继承原图」选项（置顶）。
   - **分辨率**：默认继承原图的分辨率。同样新增「继承原图」选项。
   - **质量**：默认继承原图。
   - 以上参数用户均可手动覆盖。

6. **输入补充提示词**：
   - 用户在「补充优化提示词」文本框中输入想要的修改方向。
   - 此提示词将作为最终提示词发送（不与原始提示词拼接——发送时原始提示词作为上下文，新提示词作为主要指令）。
   - **提示词拼接策略**：最终发送的 prompt = `基于参考图进行优化。原始描述：{原始prompt}。优化方向：{用户输入的补充提示词}`
   - 如果用户未输入补充提示词，使用原始提示词（意味着「保持风格，重新生成」）。

7. **点击生成优化**：
   - 请求参数中包含参考图（压缩后的 data URL）。
   - 在画布上创建新节点，位置规则：
     - 新节点放置在**父节点（选中的图片）的右侧**，水平间距 60px。
     - 如果右侧已有节点，向下偏移 `节点高度 + 40px`。
   - 新节点的 `parentId` 设为选中节点的 ID。
   - 同时创建一条 edge：从父节点指向新节点。
   - 画布自动平移，确保新节点在可视区域内。

8. **生成完成后**：
   - 右侧面板保持在优化模式（方便连续迭代）。
   - 新节点自动被选中（替代之前的选中状态）。
   - 用户可以继续基于新图再次优化（链式迭代）。
   - 画布上显示从父节点到子节点的连接线。

### 4.3 连接线（Edge）渲染

- **样式**：贝塞尔曲线，颜色 `#10B981` 50% 透明度，线宽 2px。
- **起点**：父节点右侧中点。
- **终点**：子节点左侧中点。
- **箭头**：终点处显示小三角箭头（6px）。
- **交互**：连接线不可点击、不可选中（纯视觉元素）。
- **层级**：连接线渲染在节点下方（z-index 低于节点）。

## 5. 右侧生成面板

### 5.1 面板布局

右侧面板宽度固定 360px，可通过左侧边缘的折叠按钮收起/展开。

**折叠状态**：面板完全收起，画布占满宽度。左侧边缘显示一个展开按钮（`>` 箭头图标）。

**展开状态**：面板从右侧滑入（动画 200ms ease-out）。

### 5.2 生成模式（无选中节点）

从上到下依次：

| 区域 | 内容 |
|------|------|
| 标题栏 | "画布生成" + 折叠按钮 |
| 模型选择器 | 当前选中模型名称 + 下拉菜单（只显示已验证的模型列表） |
| 提示词输入 | 多行 textarea，高度自适应（min 3行，max 8行），placeholder "描述你想生成的图片..." |
| 参数区域 | 宽高比、分辨率、质量、格式（折叠式，点击「参数」展开/收起） |
| 生成按钮 | 全宽主色按钮，文案 "生成到画布"。生成中变为 "生成中..." + 旋转图标，不可再次点击 |
| 快捷信息 | 灰色小字显示当前模型协议 + 请求尺寸预览 |

### 5.3 优化模式（选中节点后点击优化）

从上到下依次：

| 区域 | 内容 |
|------|------|
| 标题栏 | "← 返回" 按钮 + "优化图片" |
| 参考图预览 | 选中图片的缩略图（最大高度 160px），下方显示尺寸、大小、压缩状态 |
| 原始提示词 | 只读展示，灰色背景区域，最多显示 3 行，超出折叠 |
| 补充提示词 | 多行 textarea，placeholder "描述你想要的修改..." |
| 模型选择 | 下拉，默认继承原图模型 |
| 参数区域 | 宽高比（默认「继承原图」）、分辨率（默认「继承原图」）、质量 |
| 生成按钮 | 全宽主色按钮，文案 "生成优化"。生成中变为 "优化中..." |

### 5.4 面板状态切换逻辑

```
无选中节点 → 显示「生成模式」
选中一个成功节点 → 面板顶部出现「基于此图优化」入口按钮
  → 用户点击入口 or 浮动操作栏「优化」按钮 → 切换到「优化模式」
  → 用户点击「← 返回」或取消选中 → 回到「生成模式」
选中一个失败节点 → 面板显示错误详情 + 「重试」按钮
选中一个生成中节点 → 面板显示生成进度 + 「取消」按钮
```

## 6. API 调用与数据流

### 6.1 复用现有 API 配置

画布模式不新增任何后端接口。生成请求复用现有的 `/api/images/generate` 端点。

**共享的配置项**：
- `apiConfig.baseUrl` — 固定 SumAPI API URL
- `apiConfig.apiKey` — API Key
- `apiConfig.protocol` — 生图协议
- `selectedModel` — 选中模型
- 已验证的模型列表

**画布模式独立的配置项**：
- 画布自身的参数（宽高比、分辨率等）独立于 Studio 模式，互不影响。

### 6.2 生成请求构建

**自由生成**：

```typescript
{
  baseUrl: apiConfig.baseUrl,
  apiKey: apiConfig.apiKey,
  request: {
    protocol: apiConfig.protocol,
    model: selectedCanvasModel,
    prompt: userPrompt,
    referenceImages: [],
    aspectRatio: selectedAspectRatio,
    resolution: selectedResolution,
    quality: selectedQuality,
    outputFormat: "png",
    seed: "",
  }
}
```

**优化生成**：

```typescript
{
  baseUrl: apiConfig.baseUrl,
  apiKey: apiConfig.apiKey,
  request: {
    protocol: apiConfig.protocol,
    model: selectedCanvasModel,
    prompt: composedPrompt,  // 拼接后的提示词
    referenceImages: [compressedReferenceDataUrl],  // 压缩后的参考图
    aspectRatio: overriddenOrInheritedAspectRatio,
    resolution: overriddenOrInheritedResolution,
    quality: selectedQuality,
    outputFormat: "png",
    seed: "",
  }
}
```

### 6.3 参考图压缩流程

```
选中节点的原始图片（IndexedDB Blob）
  → 读取为 Image 对象
  → Canvas 绘制，最长边缩放到 1024px
  → 导出为 WebP，quality = 0.85
  → 检查大小：
      ≤ 1MB → 直接使用
      > 1MB → 降低 quality 为 0.7 重试
      仍 > 1MB → 降低 quality 为 0.5 重试
      仍 > 1MB → 缩放最长边到 768px，quality 0.7
  → 输出 data URL 作为参考图
```

该逻辑可复用 Studio 模式中已有的 `createSquareThumbnail` 或 `renderReferenceForRequest` 函数。

## 7. 画布状态持久化

### 7.1 存储策略

画布状态存储在 IndexedDB 中，与 Studio 模式的生成记录分开：

| 存储键 | 内容 |
|--------|------|
| `canvas-state` | `CanvasState` 对象（节点坐标、边、视口状态、选中状态） |
| `canvas-image-{nodeId}` | 每个节点的图片 Blob 数据 |

### 7.2 保存时机

- 节点位置变化后（拖拽结束时），防抖 500ms 后保存。
- 新节点生成成功后立即保存。
- 节点删除后立即保存。
- 视口变化后，防抖 1000ms 后保存。
- 页面 `beforeunload` 时同步保存。

### 7.3 加载策略

- 进入画布页面时，从 IndexedDB 读取 `canvas-state`。
- 逐个加载节点图片 Blob，转为 `objectUrl` 并挂载到节点对象上。
- 图片加载使用懒加载策略：优先加载视口内可见的节点图片，视口外的延迟加载。
- 加载过程中节点显示骨架屏占位。

## 11. 性能策略

### 11.1 画布渲染

- 使用 `<canvas>` 元素 或 CSS `transform` 方案渲染画布。
- **推荐方案**：使用 DOM + CSS `transform: translate() scale()` 实现。理由：
  - 项目无 Canvas 2D / WebGL 依赖，DOM 方案实现成本低。
  - 图片节点数量预期 < 200，DOM 性能足够。
  - DOM 方案天然支持文字选择、右键菜单、无障碍。
- 外层容器设置 `overflow: hidden`，内层画布层应用 `transform: translate(${x}px, ${y}px) scale(${zoom})`。

### 11.2 节点图片优化

- 缩放 < 30%：节点只渲染色块 + 状态图标，不加载图片。
- 缩放 30%~80%：使用缩略图渲染（如果有压缩版本）。
- 缩放 > 80%：使用原始图片。
- 视口外的节点图片使用 `visibility: hidden`（保留布局但不渲染）。

### 11.3 内存管理

- 每个节点的图片 Blob 在 IndexedDB 中存储。
- 运行时通过 `URL.createObjectURL(blob)` 生成 objectUrl 用于显示。
- 离开画布页面时，调用 `URL.revokeObjectURL()` 释放所有 objectUrl。
- 视口外节点的 objectUrl 可以先行释放，滚入视口时重新生成。

## 12. 边界情况与异常处理

### 12.1 API 未连接

- 右侧面板的生成按钮显示为 disabled 状态。
- 按钮下方显示提示文案："请先在配置中设置 API Key 并验证连接"。
- 提供跳转链接到 Studio 模式配置面板。

### 12.2 生成中断网

- 节点保持「生成中」状态。
- 超时（300s，与 Studio 一致）后自动标记为错误状态。
- 错误消息：「请求超时，请检查网络后重试」。

### 12.3 多节点同时生成

- 画布模式支持同时生成多个节点（无并发限制——每次点击「生成」创建一个独立请求）。
- 每个节点独立管理自己的生成状态。
- 右侧面板在生成中不阻塞——用户可以继续输入新的提示词发起新的生成。

### 12.4 浏览器存储满

- IndexedDB 写入失败时，在画布顶部显示一条警告横幅：「浏览器存储空间不足，新生成的图片可能无法保存。请下载重要图片后清理画布。」
- 提供「清理画布」按钮：按时间排序，展示节点列表，用户勾选后批量删除。

### 12.5 大量节点性能

- 节点数 > 100 时，在画布右上角显示性能提示：「画布中有 {n} 张图片，如果感觉卡顿，可以删除不再需要的图片。」
- 节点数 > 200 时，强制开启视口外节点卸载（不渲染 DOM，只保留占位矩形）。

### 13.1 与 Studio 模式的关系

| 维度 | Studio 模式 | Canvas 模式 |
|------|------------|-------------|
| 核心场景 | 批量生产，线性记录 | 迭代探索，空间化排列 |
| 生成方式 | 批量（1-20张） | 单张 |
| 参考图来源 | 手动上传 | 选中画布上的图片自动作为参考图 |
| 结果展示 | 时间线列表 | 自由位置的画布节点 |
| 数据存储 | IndexedDB（共用 DB，不同 store） | IndexedDB（共用 DB，不同 store） |
| API 配置 | 各自独立的参数面板 | 共享 API URL / Key / 模型列表 |
| 并发 | 可配置 1-6 | 无限制（每次单张） |

### 13.2 与 Square 的关系

- 画布模式生成的图片**暂不支持**推荐到广场（推荐功能依赖 Studio 模式的 Job/HistoryRecord 数据结构）。
- 后续版本可考虑增加推荐入口。

### 13.3 与 Agent 模式的关系

- 画布模式**不支持** Agent 模式（Agent 模式的批量编排逻辑与画布的单张生成范式不兼容）。
- 画布模式下 Agent 面板隐藏。

## 14. 技术实现要点

### 14.1 不新增外部依赖

画布的平移、缩放、节点拖拽使用原生 DOM 事件（pointer events）+ CSS transform 实现，不引入 react-flow、fabric.js 等外部库。理由：
- 项目已有 9700+ 行的单文件前端，控制依赖体积。
- 画布功能相对简单（只有矩形节点 + 贝塞尔连线），不需要完整的图编辑器框架。
- 自定义实现可完全控制交互细节。

### 14.2 核心 Hook 拆分

虽然在 `App.tsx` 中实现，但建议将画布逻辑组织为几个内部函数块：

- `useCanvasViewport()` — 管理平移/缩放状态和鼠标/手势事件。
- `useCanvasNodes()` — 管理节点 CRUD、拖拽、选中。
- `useCanvasGenerate()` — 管理生成请求、状态更新。
- `useCanvasPersist()` — 管理 IndexedDB 读写和防抖保存。

### 14.3 坐标系统

- **画布坐标**：节点的 `x`, `y` 是画布逻辑坐标（不受缩放影响）。
- **屏幕坐标**：实际渲染位置 = `(canvasX - viewportX) * zoom`, `(canvasY - viewportY) * zoom`。
- **交互坐标转换**：鼠标事件的 `clientX/Y` 需要转换为画布坐标：`canvasX = clientX / zoom + viewportX`。
